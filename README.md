- Oque é Levantamento de Requisitos 
   - Transformar as necessidades do negócio em informações técnicas

- Requisitos Funcionais e Não Funcionais 
   - RF - O que o sistema vai fazer
   - RNF - Como o sistema vai fazer

- Requisitos, Regras, Restrições 
    - Requisitos: A Parte Técnica do sistema
    - Regras: As Necessidade dos Clientes 
    - Restrições: Limitações impostas pela, linguagem, leis, normas

- Modelo de Documentação Requisitos

|Campo|Conteúdo|Exemplo|
|----|-------|-----|
| ID | Código | RF-001 |
| Título | Valor | Fazer Tela de Login |
| Descrição | O Ator deve ser Capaz de... | O Cliente conseguirá fazer Login |
| Prioridade | Valor da Prioridade | Alta |
| Regras Vinculadas | Outras Regras Relacionada | RN-001 - Banco de Dados |

### Técnicas de Levantamento de Requisitos
### Briefing:
Coleta de dados ou informações iniciais, realizada com o cliente, que será a base para o desenvolvimento inicial do projeto. Essa coleta ocorre em uma reunião com o cliente, chamada reunião de briefing.
    - Manual de Identidade Visual (Cores, Logo, Slogan, Fontes...)
    - Concorrentes
    - Prazos
    - Orçamento
    - Identidade da Empresa (Missão, Visão, Valores)
    - Público-Alvo 

### Brainstorm/Brainstorming:
Técnica utilizada para levantar sugestões e ideias, a fim de solucionar problemas no desenvolvimento de produtos e serviços.
    - Podem ser Estruturados ou não-Estruturados
    - Reunião Curta com Foco Bem Definido
      - Miro 
      - Mentimeter
      - OneNote   

### Questionário:
Lista de perguntas sobre os requisitos do projeto. Diferentemente do briefing, que faz coleta de dados sobre o produto ou serviço a ser desenvolvido, o design, a empresa, etc., aqui se faz perguntas sobre a implementação do produto ou serviço.
     - Formas de Fazer o Questionário
       - Reunião com Perguntas Diretas (Qualitativas)
       - Formulário de Coleta de Dados (Quatitativas)

### Entrevista:
Técnica para coleta de dados de usuários de um produto ou serviço para identificação de problemas, a partir de suas observações.
     - Entrevistas podem ser Formais e Informais
     - Nessa Etapa é necessário um MVP (Produto Mínimo Viável)

### Etnografia:
Técnica de observação utilizada para analisar requisitos sociais e culturais das tarefas desempenhadas numa organização.
     - Observação da Cultura da Empresa, a forma como ela atende e se relaciona com seu público interno e externo.

### Workshop:
Reunião composta pela equipe de analistas da empresa contratada para desenvolvimento do produto ou serviço e por representantes do cliente, a fim de especificar as necessidades dos projeto.
     - Eventos, Feiras e Workshops de produto específicos.

### Prototipagem:
Modelo preliminar ou inicial do produto que apresenta algumas funcionalidades que ele terá.
    - Criação de um elemento de design para simulação de um projeto, com intuito de testar funcionalidades e elementros gráficos antes de começar a escrever código
    - Os protótipos podem ser de:
      - Baixa fidelidade (rafe): rascunho de um desenho pode ser feito a mão
      - Média fidelidade (rough): desenhos para a marcação de espaços e layouts, não precisa de texto nem de cores
      - Alta fideliade: protótipo focado nas aplicações de funcionalidade iguais aos do projeto que serão desenvolvidos posteriormente em código
